Виконання лабораторної роботи
Рівень «задовільно»

На даному рівні було реалізовано:

отримання списку користувачів:

GET /api/v1/users

отримання одного користувача:

GET /api/v1/users/:id
відображення даних на фронтенді;
обробка станів інтерфейсу:
завантаження;
відсутність даних;
помилка.

Фронтенд отримує дані з бекенду та відображає їх у таблиці.

Рівень «добре»

На цьому рівні було реалізовано повний CRUD для сутності Users:

створення:

POST /api/v1/users

редагування:

PUT /api/v1/users/:id

видалення:

DELETE /api/v1/users/:id

Також:

реалізовано взаємодію фронтенду з API;
створено універсальну функцію apiRequest() для виконання HTTP-запитів;
після кожної операції список автоматично оновлюється.
Рівень «відмінно»

На даному рівні було розширено функціональність системи:

1. Версіонування API

Усі маршрути переведено у формат:

/api/v1/...
2. Робота зі зв’язаними даними

Реалізовано маршрут:

GET /api/v1/passes/full

який повертає:

ПІБ користувача;
роль;
причину пропуску.
3. Розділення логіки endpoint

У фронтенді для сутності Passes:

endpoint: "passes",
listEndpoint: "passes/full"

Це дозволяє:

використовувати різні джерела даних;
не дублювати код.
4. Фільтрація даних

Реалізовано фільтр пропусків за статусом:

<select id="statusFilter">
  <option value="">Усі статуси</option>
  <option value="approved">Погоджено</option>
  <option value="pending">Очікує</option>
  <option value="rejected">Відхилено</option>
</select>
if (state.statusFilter) {
  items = items.filter(
    (item) => item.status === state.statusFilter
  );
}
5. Повна інтеграція фронтенду і бекенду

Реалізовано:

синхронізацію всіх сутностей;
коректну обробку помилок;
оновлення інтерфейсу після кожної дії.

Запуск проєкту

Встановлення залежностей:

npm install

Запуск серверу:

npm run dev

Сервер буде доступний за адресою:

http://127.0.0.1:3000

Фронтенд відкривається через:

frontend/index.html
Висновок

У результаті виконання лабораторної роботи було створено повноцінний веб-додаток для керування пропусками з використанням REST API.

Було реалізовано:

взаємодію клієнта та сервера;
CRUD-операції;
роботу зі зв’язаними даними;
фільтрацію;
версіонування API.

Функціональність системи відповідає вимогам рівнів «задовільно», «добре» та «відмінно».
