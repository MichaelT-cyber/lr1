# Лабораторна робота №2 – Backend API

## Опис проєкту

Система пропусків у комп’ютерний клас.

Серверна частина застосунку реалізована за допомогою **Node.js** та **Express.js**.  
Сервер надає REST API для керування користувачами та пропусками.

## Використані технології

- Node.js
- Express.js
- REST API
- Thunder Client (для тестування API)

## Сутності системи

У системі реалізовано дві основні сутності.

### Users (Користувачі)

Представляє користувачів комп’ютерного класу.

Поля:
- id
- fullName
- role
- isActive

### Passes (Пропуски)

Представляє пропуски для доступу до комп’ютерного класу.

Поля:
- id
- userId
- validFrom
- validTo
- status

## API маршрути

### Users

GET `/api/users` — отримати список усіх користувачів  
GET `/api/users/:id` — отримати користувача за ідентифікатором  
POST `/api/users` — створити нового користувача  
PUT `/api/users/:id` — оновити дані користувача  
DELETE `/api/users/:id` — видалити користувача  

### Passes

GET `/api/passes` — отримати список усіх пропусків  
GET `/api/passes/:id` — отримати пропуск за ідентифікатором  
POST `/api/passes` — створити новий пропуск  
PUT `/api/passes/:id` — оновити дані пропуску  
DELETE `/api/passes/:id` — видалити пропуск  

## Запуск проєкту
npm install

Запуск сервера:

node src/index.js

Сервер буде запущено за адресою:

http://localhost:3000
